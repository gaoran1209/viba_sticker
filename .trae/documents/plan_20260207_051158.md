### 核心改造方案：GitHub Actions + Docker

#### 1. 自动化部署流水线 (CI/CD)
我们将利用 **GitHub Actions** 实现“代码推送即部署”。
*   **创建 `.github/workflows/deploy.yml`**：
    *   监听 `main` 分支的推送事件。
    *   通过 SSH 远程连接到您的 EC2 服务器。
    *   执行自动化脚本：拉取最新代码 -> 构建 Docker 镜像 -> 重启服务。
    *   **优势**：完全替代 Railway，利用现有的 EC2 资源，零额外成本。

#### 2. 容器化隔离 (Docker)
*   **创建 `Dockerfile` & `docker-compose.yml`**：
    *   将 Bot 封装在独立的 Docker 容器中运行。
    *   **零冲突保证**：由于 Discord Bot 主要是对外发起连接（Outbound），不需要占用服务器的入站端口（如 80/443），因此 **绝不会** 与 OpenClaw（默认端口 18789）发生端口冲突。
    *   Docker 会隔离 Python 环境，不污染服务器现有的环境。

#### 3. 您需要配合的操作 (将在文档中详细指导)
*   **服务器端**：在 EC2 上进行一次性的项目克隆 (`git clone`)。
*   **GitHub 端**：在仓库设置中添加 SSH 密钥和服务器 IP (`SECRETS`)，用于授权 GitHub Actions 登录您的服务器。
